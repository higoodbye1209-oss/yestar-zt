1. 项目概述

1.1 项目背景

艺星医疗美容集团在全国拥有多家旗舰院区，为院内会员提供高端医美服务。在诊疗及术后恢复期间，会员有院内用餐需求。目前院内点餐依赖人工口头传递，存在以下痛点：

  • 点餐效率低，会员需反复与护士沟通饮食需求
  • 术后饮食禁忌（过敏源、饮食偏好）无法系统化管理
  • 后厨备餐与配送环节缺乏数字化追踪
  • 管理层无法实时查看各院区订单数据与损耗情况

1.2 产品定位

艺星·臻味 是一款面向艺星医美院内会员的微信小程序点餐系统，同时配套 B 端管理后台。产品秉承"高端、克制、温润"的设计理念，以勃艮第酒红与暗香槟金为品牌色，打造符合医美空间调性的数字化用餐体验。

产品形态： C端微信小程序（375×812）+ B端管理后台（1280×860）
目标用户： 艺星医美院内会员、后厨人员、配送人员、院区管理层
核心场景： 术后会员在病房内通过小程序浏览菜单、下单，后厨接单备餐，配送员送餐到房

2. 产品目标

维度目标衡量指标
会员体验会员可自主完成点餐全流程点餐完成率 ≥ 90%

角色端身份核心需求

陪护人 C 端 会员家属/陪护 代会员点餐，需通过身份核验
后厨 B 端 院内厨房员工 查看新订单、开始备餐、标记完成
配送员 B 端 院内配送人员 查看待配送订单、开始配送、标记送达

权限设计： B 端采用角色化视图。后厨角色仅可见订单管理 + 配送管理 + 餐品管理；配送角色仅可见订单管理 + 配送管理；管理员角色可访问全部 8 个模块。管理员额外具备医院切换器，可按"全部医院/上海旗舰院/北京朝阳院/杭州西湖院/成都锦江院"维度筛选数据。

5. C端功能详述（会员端）

5.1 首页点餐

页面 ID：page-home

会员完成身份核验后进入首页。页面顶部展示会员问候语（"下午好，王女士"），下方为 Banner 轮播区展示今日推荐套餐。主体区域以卡片列表形式展示可点餐品，每张卡片包含餐品图片、名称、描述、价格和标签（如"术后修护""会员专享"）。

对于会员过敏源匹配的餐品，卡片显示为置灰不可点状态，点击后弹出过敏提示 Toast。右下角设有金色浮动购物车按钮，仅在购物车有商品时显示，显示商品数量角标。

触发元素目标说明
右上角头像→ 我的跳转个人中心
Banner「今日推荐套餐」→ 套餐详情进入推荐套餐页
餐品卡片→ 对应详情页跳转餐品详情
置灰餐品卡片→ Toast过敏源提示
浮动购物车按钮→ 购物车有商品时才显示
底部 Tab「订单」→ 订单列表
底部 Tab「我的」→ 个人中心

5.2 套餐推荐 & 餐品详情

C 端共含 1 个套餐推荐页 + 4 个餐品详情页。套餐推荐页展示套餐组合内容与总价，提供「一键加入购物车」按钮，点击后自动将套餐内 3 件商品加入购物车并跳转至购物车页面。餐品详情页展示单品大图、名称、描述、价格、营养成分等信息，提供「加入购物车」按钮，点击后加入购物车并弹出 Toast 提示，停留在当前页。

页面触发元素目标说明
今日推荐套餐「一键加入购物车 ¥384」→ 购物车自动加 3 件套餐
各餐品详情「加入购物车」Toast 提示加购后留在当前页
全部左上角返回箭头→ 首页点餐

5.3 购物车 & 下单结算

购物车页面展示已选商品列表，支持数量「+/-」调整（数量减至 0 时删除该商品）。底部展示合计金额与「去结算」按钮。确认下单页展示订单明细、配送地址（会员房号），提供「确认下单」和「返回修改」两个操作。确认下单后清空购物车并生成订单号，跳转至下单成功页。

页面触发元素目标说明
购物车「-」按钮减少数量/删除数量减至 0 删除
购物车「+」按钮增加数量
购物车「去结算」→ 确认下单
确认下单「确认下单」→ 下单成功清空购物车
确认下单「返回修改」→ 购物车
下单成功「查看订单」→ 订单列表
下单成功「返回首页」→ 首页点餐

5.4 订单管理

订单列表页面展示会员全部历史订单，以卡片形式按时间倒序排列。订单卡片显示订单状态（配送中/已完成）、餐品摘要、金额、时间。点击配送中订单进入进行中订单详情页，可查看配送进度、联系客服、确认收货。点击已完成订单进入详情页，可查看详情、评价、再来一单。

页面触发元素目标说明
订单列表配送中订单卡片→ 进行中订单详情

进行中详情「联系客服」Toast 提示模拟
已完成详情「评价」→ 评价页

评价页星星 1-5点亮星级留在当前页

5.5 个人中心 & 设置

「我的」页面为会员个人中心入口集合，包含以下子功能：

  • 用餐记录：展示历史用餐明细
  • 过敏源管理：四类过敏源开关（麸质/花生/海鲜/牛奶），开启后系统自动过滤相关餐品
  • 饮食偏好：四类偏好选择（清淡/低盐/低糖/高蛋白），可多选
  • 常用房号：点击显示当前绑定房号
  • 陪同点餐：管理陪护人列表，支持添加/删除陪护人

5.6 身份核验

页面 ID：page-verify

会员首次进入小程序需完成身份核验。页面展示会员姓名、房号等脱敏信息，会员确认无误后点击「确认无误，开始点餐」进入首页。核验完成前，底部 Tab「订单」和「我的」不可用，点击弹出 Toast「请先完成身份核验」。

6. B端功能详述（管理后台）

6.1 仪表盘

管理员登录后的默认首页。顶部展示核心经营指标卡片：今日订单数、今日营收、待处理订单、配送中订单。下方展示近期订单趋势（折线图占位）和待处理订单快捷列表。支持按医院筛选数据。

6.2 订单管理

展示全量订单列表，支持列表视图和看板视图切换。看板视图按订单状态分列（待接单/备餐中/配送中/已完成），支持拖拽变更状态。列表视图每行展示订单号、会员信息、餐品摘要、金额、状态、时间。点击「详情」按钮或看板卡片打开右侧订单详情抽屉，展示完整订单信息和状态时间线。点击遮罩层或关闭按钮收起抽屉。

6.3 配送管理

聚焦配送环节的操作页面。展示待配送和配送中订单列表，提供「开始配送」「标记送达」「紧急处理」等状态流转按钮。支持「批量打印小票」功能，点击弹出 Toast 模拟打印。

触发元素目标说明
「标记送达」按钮状态变更配送中 → 已完成
「开始配送」按钮状态变更待配送 → 配送中
「紧急处理」按钮状态变更标记异常
「批量打印小票」按钮Toast 提示模拟打印

6.4 餐品管理

餐品 CRUD 管理页面。以表格形式展示全部餐品，包含名称、分类、价格、库存状态、上架状态。支持「新增餐品」按钮打开模态框，填写餐品信息（名称、描述、价格、图片、分类、过敏源标签等），保存后刷新列表。模态框支持关闭按钮、取消按钮和遮罩层点击关闭。

6.5 会员管理

展示会员列表，包含姓名、手机号（脱敏）、会员等级、常用房号、过敏源信息、饮食偏好、累计消费等字段。支持搜索和筛选。点击会员行可查看详细用餐记录。

6.6 代客下单

员工代会员下单功能。选择会员后进入模拟 C 端点餐流程，由员工协助会员完成选餐、加购、下单。适用于会员不便操作手机的场景（如术后休息中）。

6.7 数据看板

面向管理层的经营数据分析页面。包含：各院区营收对比、餐品销量排行、时段订单分布、会员消费分析等可视化图表。支持按日期范围和院区维度筛选。

6.8 损耗追踪

食材损耗记录与分析模块。后厨人员记录每日食材损耗情况（损耗类型、数量、原因），管理层可查看各院区损耗趋势和损耗率统计。

6.9 角色权限与医院切换

B 端顶部栏提供角色切换器和医院选择器。

  • 角色切换：后厨 / 配送 / 管理 三个 Tab，切换后侧边导航栏菜单项根据角色权限自动显示/隐藏，同时自动跳转至该角色的默认首页
  • 医院切换：仅管理员角色可见，下拉选择全部医院 / 上海旗舰院 / 北京朝阳院 / 杭州西湖院 / 成都锦江院，切换后全局数据按所选医院筛选
角色默认页： 后厨 → 订单管理 | 配送 → 配送管理 | 管理 → 仪表盘

7.2 设计铁律

  • 酒红用于文字强调、边框、标签，不出现在大面积背景（除 VIP 卡片）
  • 暗金用于主按钮背景和价格高亮，不用于大面积装饰
  • 零 emoji，全部使用 SVG 线型图标
  • 标题使用衬线体（宋体 / Times New Roman），正文使用无衬线
  • C 端底部 Tab Bar：高度 50px + safe-area，图标 22px，标签 10px/500
  • 所有可点击元素 ≥ 44×44px
  • 字重不超过 700
  • 卡片间留白充足，模块化设计
